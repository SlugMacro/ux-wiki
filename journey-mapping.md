# Journey Mapping

> Visualize toàn bộ trải nghiệm user — từ awareness đến loyalty.

## Overview

Journey map là visual representation of user's experience over time. Nó cho thấy touchpoints, emotions, pain points, và opportunities across toàn bộ user flow. Giúp team nhìn thấy big picture thay vì chỉ focus vào individual screens.

## Key Points

### Journey Map Components
1. **Persona** — Who is taking this journey?
2. **Scenario** — What are they trying to do?
3. **Phases** — Major stages (Awareness → Consider → Purchase → Use → Advocate)
4. **Actions** — What user does at each phase
5. **Thoughts** — What they're thinking
6. **Emotions** — How they feel (emotion curve)
7. **Pain points** — Where frustration occurs
8. **Opportunities** — Where you can improve

### Types
- **Current state**: How things are NOW (identify problems)
- **Future state**: How we WANT things to be (vision)
- **Service blueprint**: Journey map + backstage processes (for teams)

### Tips
- Base on **real data**, not assumptions
- Keep it **one page** — wall-sized poster ideal
- Focus on **emotions** — that's where insights hide
- Mark **moments of truth** — make-or-break touchpoints

## Examples

- **E-commerce journey**: Search → Compare → Add to cart → Checkout → Delivery → Review
- **SaaS onboarding**: Sign up → Setup → First value → Aha moment → Habit → Upgrade
